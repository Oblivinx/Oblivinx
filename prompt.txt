### 3.2 TypeScript Features Yang HARUS Ditambahkan (Beyond Spec)

Ini adalah fitur TypeScript yang **kamu harus kreasikan** di luar yang ada di spec:

#### a) Generic Type-Safe Collection
```typescript
// Typed collection dengan full TypeScript support
interface UserDocument extends Document {
  name: string;
  email: string;
  age: number;
  address: { city: string; country: string };
  tags: string[];
}

const users = db.collection<UserDocument>('users');

// Ini harus auto-complete dan type-safe:
const result = await users.find({ age: { $gt: 18 } }); // result: UserDocument[]
await users.update({ email: 'alice@x.com' }, { $set: { name: 'Alice' } });
```

#### b) Fluent Query Builder (Chainable API)
```typescript
// Query builder yang fully chainable, type-safe, dan immutable
const results = await db.collection<UserDocument>('users')
  .query()
  .where('age', '$gte', 18)
  .where('address.city', '$eq', 'Jakarta')
  .whereIn('tags', ['admin', 'developer'])
  .whereExists('email')
  .sort('age', 'desc')
  .sort('name', 'asc')
  .limit(20)
  .skip(0)
  .project({ name: 1, email: 1, age: 1 })
  .timeout(5000)
  .hint('age_1')           // force index hint
  .explain()               // return query plan instead
  .execute();

// Aggregate builder
const report = await db.collection('orders')
  .query()
  .match({ status: 'completed' })
  .group({ _id: '$region', total: { $sum: '$amount' } })
  .sort('total', 'desc')
  .limit(10)
  .toArray();
```

#### c) SQL-Like Query Interface (Tagged Template Literal)
```typescript
// SQL-like query menggunakan tagged template literal
const users = await db.collection<UserDocument>('users')
  .sql`SELECT name, age, email 
       FROM users 
       WHERE age > ${18} AND address.city = ${'Jakarta'}
       ORDER BY age DESC
       LIMIT ${10}`;

// SQL-like yang di-compile ke MQL pipeline di balik layar
// Support: SELECT, WHERE, ORDER BY, LIMIT, SKIP, JOIN (via $lookup), GROUP BY
```

#### d) Decorator-Based Schema (Experimental)
```typescript
// Schema definition via TypeScript decorators
@Collection('users')
@Schema({ validationLevel: 'strict' })
class User extends OblivinxModel {
  @Field({ type: 'string', required: true })
  @Index({ unique: true })
  email!: string;

  @Field({ type: 'string', minLength: 2, maxLength: 100 })
  name!: string;

  @Field({ type: 'number', min: 0, max: 150 })
  age!: number;

  @Field({ type: 'object' })
  address?: { city: string; country: string };

  @Field({ type: 'array', items: { type: 'string' } })
  tags?: string[];

  @TTLIndex({ expireAfterSeconds: 86400 })
  @Field({ type: 'date' })
  sessionExpiry?: Date;

  @Versioned()  // Enable versioned document for this model
  @Field({ type: 'number', default: 1 })
  __v!: number;
}

// Repository pattern auto-generated
const userRepo = db.repository(User);
const alice = await userRepo.findOne({ email: 'alice@x.com' });
await userRepo.save(alice);
```

#### e) Event Emitter Pattern (Observable Database)
```typescript
// Database-level event system (BUKAN change stream — ini internal hooks)
class OblivinxDatabase extends EventEmitter implements IDatabase {
  // Events yang di-emit:
  // 'connected' | 'disconnected' | 'error' | 'checkpoint' | 'compaction'
  // 'slow_query' (query > threshold) | 'corruption_detected' | 'backup_complete'
  
  db.on('slow_query', (info: SlowQueryInfo) => {
    // info: { collection, filter, duration, plan }
    console.warn('Slow query detected:', info);
  });
  
  db.on('corruption_detected', (info: CorruptionInfo) => {
    // info: { page, segment, crc_expected, crc_actual }
```

### 4.1 Versioned Document System

Setiap dokumen secara opsional dapat memiliki version history lengkap.

**Arsitektur:**
- Version metadata disimpan di Metadata Segment (Section 5, Metadata Segment)
- Setiap version menyimpan: versionNumber, txId, timestamp, authorId (opsional), changeset (diff dari versi sebelumnya), snapshot (full document jika snapshot mode)
- Mode: `'diff'` (simpan hanya perubahan, hemat space) atau `'snapshot'` (simpan full document tiap version)
- Max versions per dokumen: configurable (default: 50, `maxVersions: -1` = unlimited)
- Version pruning: otomatis hapus version lama saat maxVersions tercapai (FIFO)

```typescript
// Cara aktifkan versioning per collection
await db.createCollection('articles', {
  versioning: {
    enabled: true,
    mode: 'diff',           // 'diff' | 'snapshot'
    maxVersions: 100,       // -1 untuk unlimited
    retainFor: '30d',       // Hapus versi lebih dari 30 hari
    trackAuthor: true,      // Simpan authorId di setiap versi
    autoTag: false          // Auto-create tags saat version dibuat
  }
});

// Insert awal (version 1 otomatis)
const article = await articles.insert({
  title: 'Hello World',
  content: 'Initial content',
  author: 'alice'
});

// Update (version 2 terbuat otomatis)
await articles.updateById(article.insertedId, {
  $set: { content: 'Updated content', updatedBy: 'bob' }
});

// Ambil versi spesifik
const v1 = await articles.getVersion(article.insertedId, 1);
// Returns: { _id, __version: 1, __versionedAt: Date, __author: 'alice', ...fields }

// List semua versi
const history = await articles.listVersions(article.insertedId);
// Returns: [{ version: 1, createdAt, author, changeCount }, { version: 2, ... }]

// Diff antara dua versi
const diff = await articles.diffVersions(article.insertedId, 1, 2);
// Returns: { added: {}, modified: { content: { from: 'Initial...', to: 'Updated...' } }, removed: {} }

// Rollback ke versi tertentu (membuat versi baru dengan konten lama)
const reverted = await articles.rollbackToVersion(article.insertedId, 1);

// Tag sebuah versi (seperti git tag)
await articles.tagVersion(article.insertedId, 2, 'v1.0-published');

// Restore dari tag
await articles.restoreFromTag(article.insertedId, 'v1.0-published');
```

### 4.5 Backup & Recovery System

**Backup Modes:**
1. **Full Backup**: Copy seluruh .ovn file (hot backup, tidak perlu lock)
2. **Incremental Backup**: Hanya WAL records yang baru sejak backup terakhir
3. **Logical Backup**: Export ke format JSON/NDJSON yang human-readable
4. **Encrypted Backup**: AES-256-GCM encrypted backup dengan key derivation (PBKDF2)

**Backup Format (.ovnbak):**
```
Header: Magic (4B) + Version (2B) + Timestamp (8B) + Checksum (4B)
Metadata: JSON — originalPath, collections, size, compression, encryption
Body: Compressed (LZ4) + optionally encrypted pages
Footer: CRC32 of entire file
```

**API:**

## BAGIAN 5 — SECURITY (WAJIB LAYAKNYA DATABASE PRODUKSI)
## ═══════════════════════════════════════════════════════

Kamu WAJIB mengimplementasikan semua fitur security berikut. Database tanpa security bukan database produksi.

### 5.1 Encryption at Rest

```
Per-collection AES-256-GCM encryption (sesuai Section 5, Bit 1 Flags)

Key Management:
- Master key: 256-bit random, disimpan terpisah dari .ovn file
- Per-collection keys: Di-derive dari master key menggunakan HKDF
  collection_key = HKDF(master_key, collection_name, "oblivinx3x-col-enc", 32)
- Key rotation support: Re-encrypt dengan new master key tanpa downtime
- Key storage options:
  1. File-based: Encrypted key file (AES-256 dengan password via PBKDF2)
  2. Environment variable: OVN_MASTER_KEY (base64-encoded)
  3. Hardware: Abstract KeyProvider interface untuk HSM/KMS integration

Encryption scope:
- Document data pages: Encrypted sebelum write ke disk
- Index pages: Encrypted (index keys juga terenkripsi)
- WAL records: Encrypted sebelum flush
- Metadata segment: Encrypted
- TIDAK terenkripsi: File header page (diperlukan untuk open), Segment Directory
```

### 5.2 Page Integrity (CRC32)

```
Setiap page memiliki CRC32 checksum di Page Header (Section 5.3, 4B CRC32 field)

Pada setiap page READ:
1. Hitung CRC32 dari byte 0 s/d end_of_page (kecuali 4B CRC32 field itu sendiri)
2. Bandingkan dengan stored CRC32
3. Jika mismatch: throw CorruptionError({ page, segment, expected, actual })
4. Log ke internal audit log
5. Trigger 'corruption_detected' event

Pada setiap page WRITE:
1. Hitung CRC32 dari konten page
2. Store di Page Header sebelum flush

Integrity check API:
await db.pragma('integrity_check', 'full');
// Scan semua pages, semua indexes, semua WAL records
// Returns: { status: 'ok' | 'corrupted', issues: CorruptionIssue[] }
```

```typescript
// tests/integration/versioning.test.ts
describe('Versioned Document Integration Tests', () => {
  it('should track full version history through multiple updates', async () => {
    await db.createCollection('documents', {
      versioning: { enabled: true, mode: 'diff', maxVersions: 50 }
    });
    
    // Version 1: Initial
    const { insertedId } = await db.insert('documents', {
      title: 'My Document',
      content: 'Original content here',
      status: 'draft',
      wordCount: 3
    });
    
    // Version 2: Content update
    await db.update('documents', { _id: insertedId }, {
      $set: { content: 'Updated and improved content', wordCount: 4 }
    });
    
    // Version 3: Status change
    await db.update('documents', { _id: insertedId }, {
      $set: { status: 'published', publishedAt: new Date() }
    });
    
    // Version 4: Minor edit
    await db.update('documents', { _id: insertedId }, {
      $set: { content: 'Updated and improved content with fixes' }
    });
    
    const doc = db.collection('documents');
    
    // Verify 4 versions exist
    const history = await doc.listVersions(insertedId);
    expect(history).toHaveLength(4);
    expect(history[0].version).toBe(1);
    expect(history[3].version).toBe(4);
    
    // Verify version 1 content
    const v1 = await doc.getVersion(insertedId, 1);
    expect(v1?.content).toBe('Original content here');
    expect(v1?.status).toBe('draft');
    
    // Verify diff between v1 and v3
    const diff = await doc.diffVersions(insertedId, 1, 3);
    expect(diff.modified).toHaveProperty('content');
    expect(diff.modified).toHaveProperty('status');
    expect(diff.added).toHaveProperty('publishedAt');
    
    // Rollback to v1
    const reverted = await doc.rollbackToVersion(insertedId, 1);
    expect(reverted.content).toBe('Original content here');
    expect(reverted.status).toBe('draft');
    
    // After rollback, a new version (5) should be created
    const postRollbackHistory = await doc.listVersions(insertedId);
    expect(postRollbackHistory).toHaveLength(5);
    expect(postRollbackHistory[4].version).toBe(5);
  });
  
  it('should prune old versions when maxVersions is exceeded', async () => {
    await db.createCollection('limited_versions', {
      versioning: { enabled: true, maxVersions: 5 }
    });
    
    const { insertedId } = await db.insert('limited_versions', { value: 'v0' });
    
    // Create 10 versions (exceeds maxVersions: 5)
    for (let i = 1; i <= 10; i++) {
      await db.update('limited_versions', { _id: insertedId }, {
        $set: { value: `v${i}` }
      });
    }
    
    const history = await db.collection('limited_versions').listVersions(insertedId);
    
    // Should keep only last 5 versions
    expect(history.length).toBeLessThanOrEqual(5);
    
    // Latest should be version 11 (initial + 10 updates)
    expect(history[history.length - 1].version).toBe(11);
  });
  
  it('should handle concurrent updates with versioning correctly', async () => {
    const { insertedId } = await db.insert('documents', { counter: 0 });
    
    // Concurrent updates (MVCC should handle this)
    const updates = Array.from({ length: 10 }, (_, i) =>
      db.update('documents', { _id: insertedId }, { $inc: { counter: 1 } })
    );
    
    await Promise.allSettled(updates);
    
    const final = await db.findOne('documents', { _id: insertedId });
    // Counter harus terupdate (meski concurrent, MVCC serializes writes)
    expect(final?.counter).toBeGreaterThan(0);
  });
});
```

```typescript
/**
 * Filter dokumen berdasarkan nilai field menggunakan MQL comparison operator.
 * Method ini IMMUTABLE — selalu return QueryBuilder instance baru.
 *
 * @param field - Nama field (support dot notation, e.g. "address.city")
 * @param operator - MQL operator: '$eq' | '$ne' | '$gt' | '$gte' | '$lt' | '$lte'
 * @param value - Nilai yang dicompare. Tipe disesuaikan dengan field.
 *
 * @returns QueryBuilder<T> — Instance baru dengan filter ditambahkan
 *
 * @throws {QueryError} Jika operator tidak valid
 * @throws {TypeError} Jika field bukan string
 *
 * @example
 * // Query users dengan umur 18-60
 * const q = builder
 *   .where('age', '$gte', 18)
 *   .where('age', '$lte', 60);
 *
 * @example
 * // Query dengan nested field
 * const q = builder.where('address.city', '$eq', 'Jakarta');
 *
 * @see {@link FilterBuilder} untuk filter yang lebih complex
 * @see {@link https://docs.oblivinx.dev/query-builder#where} untuk dokumentasi lengkap
 */
where(field: keyof T | string, operator: ComparisonOp, value: unknown): QueryBuilder<T> {
  // Validasi operator menggunakan Set untuk O(1) lookup
  if (!VALID_COMPARISON_OPS.has(operator)) {
    throw new QueryError(
      `Invalid comparison operator: '${operator}'. Valid: ${[...VALID_COMPARISON_OPS].join(', ')}`,
      { field, operator, value }
    );
  }
  
  // Immutable: buat internal filter baru dengan spread
  const newFilter = {
    ...this._filter,
    [field]: { ...((this._filter as any)[field] || {}), [operator]: value }
  };
  
  // Return instance baru (bukan mutate this)
  return new QueryBuilder<T>(this._collection, newFilter, this._options);
}
```
