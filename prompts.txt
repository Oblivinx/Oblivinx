### FILE 08: ovnplanning/07-SECURITY-LAYER.md
Isi SANGAT detail untuk setiap fitur security:

**Encryption at Rest:**
- Key derivation: Argon2id(password, salt, m=65536, t=3, p=4) → 32-byte master key
- File encryption: AES-256-GCM per-page, 12-byte IV = random(4) + page_no(4) + counter(4)
- Authentication tag: 16 bytes per page untuk tamper detection
- Key rotation protocol: re-encrypt semua page dengan key baru dalam satu transaction
- Key storage: never in plaintext, support external KMS (HashiCorp Vault, AWS KMS)

**Encryption in Transit:**
- TLS 1.3 only, disable older versions
- Cipher suites: TLS_AES_256_GCM_SHA384, TLS_CHACHA20_POLY1305_SHA256
- Certificate pinning untuk embedded mode
- Mutual TLS (mTLS) untuk server-to-server

**Field-Level Encryption (FLE):**
- Per-field encryption key derived dari master key + field_path
- Deterministic vs randomized encryption: tradeoff untuk searchability
- Encrypted field metadata: type, key_id, algorithm, iv disimpan di field header

**RBAC:**
- User model: user_id, password_hash(Argon2), roles[], created_at, last_login
- Role model: role_id, permissions[], collection_patterns (glob)
- Permission types: read, write, delete, admin, backup, schema
- Attribute-based access control (ABAC) extension untuk fine-grained policy
- Session token: JWT RS256, exp: 1h, refresh token: opaque 32-byte, exp: 30d

**Row-Level Security:**
- Policy expression: AST expression evaluated per-document
- Example: `{owner_id: {$eq: "$current_user.id"}}`
- Performance impact: policy expression di-push ke query planner sebagai filter

**Audit Log:**
- Append-only log file, separate dari main DB
- Record format: timestamp, user, operation, collection, document_id, old_value_hash, new_value_hash, ip, duration_ms
- Tamper evidence: setiap record chain dengan HMAC-SHA256 dari record sebelumnya
- Retention policy: configurable, default 90 hari, auto-archive ke compressed file

**Query Injection Protection:**
- No string interpolation dalam query — parameterized only
- Input sanitization: null byte, control character stripping
- Depth limit: max 32 level nested object dalam query

**Rate Limiting:**
- Token bucket per API key: capacity, refill rate, burst limit
- Per-IP fallback untuk unauthenticated connections
- Penalty box: temporary ban setelah N failed auth attempts

**Secure Delete:**
- DOD 5220.22-M: 3-pass overwrite (0x00, 0xFF, random) sebelum free
- SSD-aware: TRIM + crypto-erase (hapus key, bukan data)

**Integrity Verification:**
- Per-page SHA-256 checksum di header page
- Merkle tree atas seluruh database untuk full integrity verification
- PRAGMA integrity_check analogue

### FILE 09: ovnplanning/08-SEARCH-ENGINE.md
Isi:
- Architecture search pipeline: Query → Tokenize → Lookup → Score → Rank → Return
- Tokenizer detail: whitespace, unicode word boundary, CJK bigram, Indonesian stemmer
- Stop words: daftar lengkap Bahasa Indonesia + English, configurable
- Stemmer: Porter Stemmer untuk English, PySastrawi-inspired untuk Indonesia
- Inverted index structure detail (lihat FILE 05 tapi lebih dalam di sini tentang search)
- Query types: simple term, phrase, prefix, wildcard, boolean, proximity, fuzzy
- Fuzzy matching: BK-tree untuk Levenshtein lookup, edit distance threshold adaptive
- Scoring formula lengkap dengan pseudocode:
  ```
  score(d, q) = sum_t[ idf(t) * tf(t,d) * fieldBoost(f) * freshnessBoost(d) ]
  idf(t) = log(1 + (N - df(t) + 0.5) / (df(t) + 0.5))
  tf(t,d) = (freq(t,d) * (k1+1)) / (freq(t,d) + k1 * (1-b+b*dl/avgdl))
  ```
- Field boosting: title^3, tags^2, body^1 — konfigurasi per collection
- Highlight generation: extract snippet 150 char dengan term highlighted
- Search suggestions: prefix trie dari top-K terms, typo correction via BK-tree
- Faceted search implementation: aggregation over search results per category bucket
- Vector + keyword hybrid search: RRF (Reciprocal Rank Fusion) untuk gabungkan hasil
